# WANDERLOST — App Workflow (page-by-page, state-by-state)

> This document is the source of truth the AI builder follows screen by screen. Nothing may be skipped.
> Companion docs: `PRD.md`, `TECH_SPEC.md`, `DESIGN_SYSTEM.md`.

**Global rules (every page):** persistent SIMULATION badge; bottom tab bar on mobile (Search, Trips, Passport, Settings); reduced-motion honored; safe-area padding; toasts top; all loading states use the departure-board / paper-plane motif; all errors are friendly and offer retry without losing data; back navigation preserves state.

## 0. App shell / layout
Nav tabs; theme; SIMULATION badge (tapping it opens a short explainer sheet); offline banner when the SW detects offline.

## 1. Landing (`/`)
- **Content:** kinetic hero (display type "All the thrill. None of the jet lag."), animated paper-plane / departure-board motif, "Start dreaming" CTA, 3-step how-it-works, ethics link.
- **States:** default; reduced-motion (static hero); returning user (CTA → "Continue" to Trips).
- **Interactions:** CTA → Onboarding (first visit) or Search (returning); hover/press micro-interactions.

## 2. Onboarding / Simulation Disclosure (`/welcome`)
- **Content:** one screen, plain language: "Everything here is a beautiful lie. No real flights, no real hotels, no payment — ever. Just the joy of planning." Illustration; "I understand — let's dream" button; optional name entry (skippable); optional "How do you feel right now?" mood check (skippable).
- **States:** default; validation (must acknowledge); reduced-motion.
- **Interactions:** acknowledge → store flag + create guest user → Search. Cannot proceed without acknowledging.

## 3. Flight Search (`/search/flights`)
- **Content:** origin & destination autocomplete, date(s) with round-trip toggle, passengers stepper, cabin select, "Search flights" CTA; "Surprise me" (Tier 3) picks a random dream route.
- **States:** empty/default (popular-destination suggestions); typing (autocomplete: airport name, city, IATA — keyboard + touch); validation (origin=destination blocked, past departure blocked, missing fields flagged); loading (immediate transition to results with departure-board loader).
- **Interactions:** field focus (plane taxi along underline); select advances focus; recent-search chips.

## 4. Flight Results (`/search/flights/results`)
- **Content:** header summarizing query (editable), sort (price/duration/departure), filters (stops, cabin, airline, time-of-day), list of FlightResultCards (airline mark, times, duration, stops, price, "you'd save $X").
- **States:** loading (**departure-board split-flap** signature animation); success (staggered card entrance); empty/no-results ("No flights on this route — try nearby airports"; rare); error (retry card).
- **Interactions:** tap card → shared-layout transition to detail; filter/sort re-runs the seeded engine (stable results); lazy list; swipe between cards on mobile.
- **Edge cases:** same-day return; very short/long routes; island/remote airports (engine still returns a plausible 1-stop).

## 5. Flight Detail / Selection (`/flight/[id]`)
- **Content:** full itinerary breakdown (segments, layovers, aircraft flavor, baggage flavor text), price + savings, fare-class options, "Select this flight" CTA.
- **States:** default; selected (persists to trip draft); loading; error.
- **Interactions:** select → hotel search or trip summary; back preserves results scroll.

## 6. Hotel Search (`/search/hotels`)
- **Content:** destination (prefilled from flight), check-in/out (prefilled from trip dates), guests/rooms, "Search hotels" CTA; option to skip hotels.
- **States:** default; validation (checkout > checkin); loading; skip.
- **Interactions:** prefilled from flight selection; editable.

## 7. Hotel Results (`/search/hotels/results`)
- **Content:** HotelCards (generated name, star rating, neighborhood, nightly + total price, savings, seeded illustrative hero, amenity chips), sort/filter (price, stars, amenities).
- **States:** loading (paper-plane/board), success (stagger), empty (unlikely; suggestions), error.
- **Interactions:** tap → hotel detail; filters re-run the seeded generator.

## 8. Hotel Detail (`/hotel/[id]`)
- **Content:** gallery (seeded illustrations), description flavor text, amenities, map placeholder (static, labeled illustrative), room types, price + savings, "Select hotel" CTA.
- **States:** default; selected; loading; error.

## 9. Trip Summary (`/trip/summary`)
- **Content:** selected flight(s) + hotel, dates, travelers, total "real" price and total savings, editable autogenerated trip title, "Continue to checkout."
- **States:** complete; missing pieces (prompt to add or skip); loading.
- **Interactions:** edit any segment (returns to relevant search preserving state); rename trip.

## 10. Fake Checkout (`/trip/checkout`) — critical
- **Content:** order summary; price line shows the struck-through synthetic total and **$0.00** due; big "You're saving $X" reframing; pinned SIMULATION banner ("No payment. No card needed. This isn't real — enjoy it."); **absolutely no payment/card/billing fields**; optional email field ("Send my boarding pass?"); "Confirm booking" CTA.
- **States:** default (ready); submitting (server action creates trip+booking+stamp); success (→ boarding-pass reveal); error (friendly retry; never loses the trip draft).
- **Interactions:** Confirm → **boarding-pass reveal set-piece** (GSAP + confetti + stamp). Reduced-motion: instant pass + static celebratory graphic.
- **Hard rules:** no payment fields; SIMULATION always visible; email optional and clearly for the pass only.

## 11. Confirmation / Boarding Pass (`/trip/confirmation/[bookingId]`)
- **Content:** on-screen BoardingPass (route, fictional airline, flight no, date, seat, gate, class, simulated QR, SIMULATION watermark), "Download PDF," "Add to My Trips" (auto), "Start the countdown," share (opt-in). Confirmation email sent if provided.
- **States:** success (default); PDF generating (button spinner); email sent/skipped note; error (pass always viewable in-app even if PDF/email fail).
- **Interactions:** download → `/api/boarding-pass/[id]`; passport stamp already applied (link to passport); CTA to countdown.

## 12. My Trips Dashboard (`/trips`)
- **Content:** grid/list of TripCards grouped by status (Upcoming with mini-countdown, In progress, Memories), savings-tally header, "Plan another" CTA.
- **States:** empty (illustrated "Your adventures await" + CTA); populated (cards with countdown chips); offline (cached trips readable, "offline" badge); loading/error.
- **Interactions:** tap trip → detail; overflow → rename/delete (confirm, no confirm-shaming).

## 13. Trip Detail + Countdown (`/trips/[tripId]`)
- **Content:** hero (seeded destination art), **CountdownFlip** to departure, trip facts, links to Itinerary, Packing, Boarding pass, Passport stamp, savings, **Share** (opens WF §20b) and **Add to calendar** (WF §20d). Collaborator avatars shown if the trip is shared. After end date → Memories surfaces.
- **States:** upcoming (live countdown); departure-day/in-progress ("Bon voyage!" celebratory state); past/memory (Memories unlocked banner); loading/offline/error.
- **Interactions:** seconds tick (reduced-motion: static with periodic update + aria-live on day change); navigate to sub-pages.
- **Edge case — past-dated trips:** if a trip's date already passed, skip gracefully to the Memory state; never show a negative countdown.

## 14. Itinerary Builder (`/trips/[tripId]/itinerary`) — Tier 2
- **Content:** day-by-day list/columns, add item (title, time, category, note), drag-to-reorder, **"Add from Explore"** (opens the Explore sheet, WF §21, to insert real POIs into a day), optional **"AI suggest itinerary"** (LLM) proposing destination-specific items (which may reference seeded POIs), **"Export itinerary"** (share/print-friendly view or PDF — usable for real trips).
- **States:** empty (prompt + AI suggest); populated; AI loading (board/plane loader); AI error/quota-exceeded (graceful "try again later," manual still works); saving (optimistic); offline (queue).
- **Interactions:** touch-friendly drag reorder; inline edit; delete with undo toast; AI accept/edit.

## 15. Packing List (`/trips/[tripId]/packing`) — Tier 2
- **Content:** auto-generated starter list by destination climate + trip length + categories (clothing, docs, tech, toiletries); check-off; add custom; progress bar.
- **States:** empty→auto-fill; populated; all-checked celebration (small); saving/offline.
- **Interactions:** satisfying tick micro-interaction; add/remove; reset.

## 16. Virtual Passport (`/passport`)
- **Content:** passport cover + pages of collected StampCards (country/city/date/style), map of "visited" places, counts (countries, cities, miles dreamed), savings total.
- **States:** empty (blank passport + "your first stamp awaits"); populated (stamp grid); stamp detail on tap; loading/offline.
- **Interactions:** tap stamp → detail sheet; new stamps animate in when arriving from a booking.

## 17. Trip Memories (`/trips/[tripId]/memories`) — Tier 2
- **Content:** auto-generated recap after the fake end date — a "postcard," the route, the stamp, savings, an optional reflective note field, optional LLM-generated summary of the imagined trip.
- **States:** locked (before end date — shows unlock date); unlocked (recap); loading; error.
- **Interactions:** add note/photo (optional, local); share postcard (opt-in).
- **Edge case:** end date passes while app is open → unlock gracefully with a gentle notification.

## 18. Settings (`/settings`)
- **Content:** profile (name, optional email/upgrade to account), **reduced-motion toggle**, theme (light/night flight), notifications opt-in (web push) + frequency, mood check-in on/off, **data export (JSON)**, **delete all data** (plain confirm), About/Ethics link, credits (OurAirports).
- **States:** default; saving; export generating; delete confirm; guest vs account (upgrade CTA).
- **Interactions:** toggles apply immediately; delete cascades and returns to landing.

## 19. About / Ethics (`/about`)
- **Content:** why this exists (plain-language summaries of Nawijn 2010, Kumar/Killingsworth/Gilovich 2014, Schultz), the ethics pledge (no payments, no dark patterns, data minimalism), data credits (OurAirports public domain), FAQ ("Is any of this real?" → no).
- **States:** static; reduced-motion.

## 20. Explore Destination (`/explore/[city]` and `/trips/[tripId]/explore`) — Tier 2 (REAL places)
- **Content:** destination header (seeded art + real city name), category tabs (Sights · Museums · Food & Cafés · Viewpoints & Parks), PoiCards — real name, short description, distance from city center, category icon, image (when licensed), **"Real place" chip**, source attribution line. "Add to itinerary" button on each card (picks a day via sheet). Attribution footer ("© OpenStreetMap contributors; Wikidata; Wikivoyage CC BY-SA").
- **States:** loading (paper-plane loader); populated (staggered cards); city not yet seeded ("We haven't mapped this dream yet" + suggest nearby seeded city); offline (cached POIs for booked trips readable); error (retry).
- **Interactions:** tab switch (chip filter); tap card → POI detail sheet (full description, map-style static locator labeled illustrative, attribution, "Add to itinerary"); add → day-picker sheet → success toast with undo; entry points: trip detail ("Explore {city}"), itinerary builder ("Add from Explore"), and hotel/flight detail pages ("What's there?").
- **Hard rules:** POIs never show fake prices, ratings, or reviews; "Real place" chip always present; attribution always visible on detail; any simulated booking content nearby keeps its SIMULATION label. This page makes fake trips useful for real planning — pair with "Export itinerary" (WF §14).

## 20b. Share Sheet (from Trip Detail / Itinerary) — Tier 2
- **Content:** "Share this dream" sheet — permission selector (**View only** / **Can add ideas**), generated link with Copy button, native share (Web Share API) on mobile, list of active links (created date, view count, Revoke), list of collaborators (name, role, downgrade/remove).
- **States:** no link yet (Generate CTA); link generated (copy + share); copied (toast); revoked (row greys out, link dies immediately); rate-limited (friendly "you've shared a lot today"); error/retry.
- **Interactions:** generate → token created; permission change updates existing link; revoke asks a plain confirm (no shaming); remove collaborator is reversible by re-sharing.
- **Hard rules:** sharing is never required to unlock anything; no "invite N friends" prompts anywhere; the sheet never suggests posting publicly.

## 20c. Shared Trip View (`/s/[token]`) — public, no account needed
- **Content:** destination art, trip title + owner display name ("Priya's dream trip"), dates, route summary, day-by-day itinerary (with "added by" chips), Explore POIs referenced, **prominent SIMULATION banner** ("This is a dream trip — nothing here is booked or paid for"), and a gentle "Make your own" CTA.
- **States:** view-only (default); editable-invite (shows "Add your ideas" CTA → sign-in/guest-upgrade → becomes collaborator); revoked/expired ("This dream has been tucked away"); trip deleted (same friendly page); loading; offline (cached if previously opened).
- **Interactions:** viewer with edit permission can add/edit/reorder itinerary items and add POIs from Explore — **only** the itinerary; bookings, trip deletion, passport, savings, and the owner's other trips are never accessible or visible.
- **Edge cases:** owner revokes while a collaborator has the page open → next mutation fails gracefully with an explanatory message; collaborator's added items remain (attributed) unless the owner deletes them.
- **Hard rules:** SIMULATION label must be visible without scrolling; never expose the owner's email; no viewer tracking beyond an aggregate count for the owner.

## 20d. Add to Calendar (from Trip Detail, Confirmation, and Shared Trip)
- **Content:** "Add to calendar" button → downloads `.ics` (works with Google, Apple, Outlook). Optional checkbox "include itinerary days as all-day events."
- **States:** default; generating (button spinner); downloaded (toast: "Saved — it'll show as *Free* so no one thinks you're actually away"); unsupported/blocked download (fallback: copy link to file); error/retry.
- **Interactions:** single tap download; on mobile the OS opens the system calendar importer.
- **Hard rules (real-world confusion guards):** event title self-labels as pretend (`✈ (Pretend) {City} · Wanderlost`); description opens with the simulation disclaimer; availability is **Free** (`TRANSP:TRANSPARENT`), never Busy; status TENTATIVE; no attendee invites are sent; no Google OAuth or calendar read access is ever requested.

## 21. Global states
- **Offline (PWA):** app shell + cached trips/passport available; search shows "you're offline — dreams need a connection" with cached suggestions; new saves queue and sync.
- **Errors:** friendly copy, retry, never dead-ends; trip drafts never lost.
- **Empty states:** every list has an illustrated, encouraging empty state.
- **Install prompt:** contextual, non-nagging, dismissible; only after a delightful moment (e.g., first boarding pass).
